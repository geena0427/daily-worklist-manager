# Daily Worklist Manager

> 把零散工作输入，变成一个可持续运转的个人工作系统。

Daily Worklist Manager 是一套基于 Markdown 文件持久化的 workflow skill。  
它不是简单的待办清单，而是一个能持续接住你的零散输入、维护长期记忆、动态重排优先级，并生成每日行动建议的个人工作管家。

## 它能做什么

- **接住零散输入**：随时记录背景、事项、状态更新
- **维护长期记忆**：区分背景与任务，持续更新任务池
- **动态评估优先级**：根据背景变化不断刷新任务排序
- **输出每日计划**：自动整理“今日必做 3 件事”和“挑战完成 3 件事”
- **继续补位推进**：当你还有精力时，给出下一件最适合做的事
- **支持长期使用**：活跃任务、归档任务、索引、快照、变更日志分层维护

---

## 它解决什么问题

真实工作里，很多信息并不是以“标准待办”的形式出现的，而是：

- 领导一句新的要求
- 一个临时想到要跟进的事项
- 一个正在推进但还没结束的任务
- 一个会影响优先级判断的背景变化
- 一个今天不一定做，但不能忘的动作

这些内容如果只散落在聊天记录、备忘和脑海里，很快就会变得混乱：

- 任务容易漏
- 优先级容易过期
- 背景变化不会同步影响待办
- 没明确结束的事项会悄悄消失
- 第二天又要重新整理一遍脑子里的工作

Daily Worklist Manager 的目标，就是把这些零散输入沉淀成一套真正能持续运转的工作系统。

## 它和普通待办工具有什么不同

### 1. 它不只记录事项，还记录“判断依据”

很多任务为什么重要，不只是因为它本身，而是因为它背后有背景。

例如：

- 当前领导更强调聚焦本职职责
- 当前阶段更重视可规模化推广的事情
- 某类工作现在优先级显著上升
- 某个方向暂时不值得投入

所以它会把信息拆成两类：

- **背景**
- **具体事项**

背景不会混进待办列表里，但会持续影响任务排序。

### 2. 它默认“用户不确认，任务不结束”

除非用户明确说：

- 做完了
- 不做了
- 放弃了

否则任务不会被自动关掉。  
这意味着没被明确结束的任务，第二天仍然会继续进入待办池参与排序。

这样做的好处是：

- 更稳
- 不容易漏事
- 不会因为 AI 猜测就把任务悄悄关闭

### 3. 它是长期记忆系统，不是一次性清单

它会持续维护：

- 背景记忆
- 活跃任务
- 归档任务
- 任务索引
- 今日视图
- 输入日志
- 变更日志与快照

这让它不只是“帮你临时列一次清单”，而是能随着你每天的输入持续更新。

### 4. 它为长期使用做了结构优化

为了避免任务越来越多、Agent 越来越慢，Daily Worklist Manager 将任务拆成三层：

- `tasks_active.md`：当前活跃任务，参与日常排序
- `tasks_archive.md`：已完成或已放弃的历史任务
- `tasks_index.md`：轻量索引，帮助快速理解全局状态

这样既保留长期记忆，又不会让每次日常操作都变得很重。

## 核心设计原则

Daily Worklist Manager 的设计围绕以下原则展开：

### 背景与事项分开
背景用于影响判断，事项用于推进执行。

### 只有用户确认，任务才进入结束态
避免任务被误判“已完成”或“已失效”。

### 日常高频操作优先轻量
新增事项、更新 doing / blocked、查看 today list 应尽量快。

### 高风险操作优先可恢复
归档、回滚、批量修改要保留快照与变更日志。

### 长期记忆必须可审阅
所有关键内容都应落到 Markdown 文件中，而不只停留在对话里。

## 核心文件说明

### `SKILL.md`
skill 的执行规则和工作流定义。  
Agent 应按它来理解文件结构、更新逻辑、快路径 / 慢路径和恢复规则。

### `memory/background.md`
记录背景总结，用于影响优先级判断。

### `memory/tasks_active.md`
记录当前活跃任务，参与日常优先级排序。

### `memory/tasks_archive.md`
记录已归档的历史任务，例如已完成或已放弃事项。

### `memory/tasks_index.md`
记录轻量总览，帮助快速理解当前全局状态。

### `memory/today.md`
存放最新一次生成的每日优先级结果。

### `memory/inbox.md`
记录用户原始输入，便于追溯。

### `memory/changes.md`
记录高风险操作后的变更日志，便于恢复。

### `snapshots/`
存放关键修改前生成的快照文件。

## 使用方式

### 1. 记录背景

适合输入组织变化、领导要求、阶段重点等内容。

示例：

```text

### 2. 记录新增事项

适合输入你刚想到的、需要跟进的、需要推进的具体任务。

示例：

```text

我想到一个事：要确认合作边界。

### 3. 更新任务状态

适合在任务开始、卡住、完成、放弃时使用。

```text

这个做完了。
这个先不做了。
